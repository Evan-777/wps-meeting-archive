# v0.1.1

Compatibility release for the `authorize-user` flow.

## Highlights

- `authorize-user` is now compatible with older `config.json` files
- Missing `redirect_uri` now defaults to `http://127.0.0.1:8765/callback`
- Missing `scope` now falls back to the built-in meeting/contact scopes

## Why this release

Some existing local configs only stored `access_token`, `client_id`, and `client_secret`.
After the CLI-first release, those older configs could no longer run `authorize-user`
without manually adding `redirect_uri` and `scope`.

`v0.1.1` restores a smoother upgrade path:

- old configs can authorize again without manual JSON surgery
- users can refresh their user token and continue syncing meetings normally
