# v0.1.2

Small compatibility release for Windows launcher scripts.

## Highlights

- Windows `.bat` launchers now try `py` first and automatically fall back to `python`
- Clearer error message when neither `py` nor `python` is available in `PATH`
- README updated to document the Windows launcher fallback behavior

## Why this release

Some Windows machines do not have the Python launcher (`py`) installed, even though
Python itself is available as `python`.

`v0.1.2` makes the double-click launchers more robust:

- works on machines with `py`
- also works on machines that only provide `python`
- gives a clearer prompt when Python is not installed at all
